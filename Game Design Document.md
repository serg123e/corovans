# КОРОВАНЫ — Game Design Document

*Документ описывает фактическое состояние игры по текущему коду репозитория. Все цифры взяты из `js/utils.js` (`CONST`) и игровых модулей.*

---

## 1. Концепция

**Жанр:** аркадный top-down экшен, волновой выживач.
**Платформа:** браузер, HTML5 Canvas + vanilla JS (ES modules), без сборки и ассетов.
**Сеттинг:** пустыня, дорога, купеческие корованы и их охрана.
**Роль игрока:** одинокий разбойник, который нападает на проходящие по дороге корованы, убивает охрану, грабит золото и прокачивается между волнами.
**Слоган:** *«грабь корованы!»*

**Core loop:**
1. Волна: несколько корованов двигаются по дороге под охраной.
2. Игрок убивает стражу, ломает повозки, собирает золото.
3. По завершении волны — магазин апгрейдов.
4. Следующая волна сложнее; каждая 5-я — босс.
5. Смерть игрока → Game Over со счётом.

**Уникальные черты:**
- Всё рисуется кодом — ни одного спрайта/звукового файла. Пиксель-арт собирается из символьных карт, звук синтезируется через Web Audio API.
- Плавное управление одновременно с клавиатуры, мыши (включая «удерживай ЛКМ/ПКМ, чтобы идти к курсору») и тача.
- Процедурная пустыня и изогнутая дорога через Catmull-Rom.

---

## 2. Управление

**Клавиатура:**
- `WASD` / стрелки — движение (нормализованный 8-направленный вектор).
- `Space` — атака (срабатывает на отпускание).
- `Shift` (левый или правый) — рывок в направлении движения (или facing, если стоим). 0.18 с, ~460 u/s, c i-frames 0.22 с, кулдаун 1.0 с.
- `Esc` / `P` — пауза.
- `M` — mute.
- `Enter` — «пропустить волну» в магазине.

**Мышь:**
- `ЛКМ` (клик) — атака в сторону курсора (персонаж доворачивается).
- `ЛКМ` (удержание) — идти к курсору.
- `ПКМ` (удержание) — идти к курсору без атаки (контекстное меню браузера блокируется).
- Вокруг игрока мёртвая зона 18 px — чтобы персонаж не дёргался у самого курсора.

**Тач:**
- Тап короче 220 мс и смещением < 20 px — трактуется как атака.
- Перетаскивание — движение в направлении пальца относительно экранной позиции игрока (та же 18-px мёртвая зона).
- Мультитач игнорируется.

**Меню/Game Over:** любой клик / `Space` / `Enter` возвращает в главное меню или запускает игру.

## 3. Архитектура игры

### 3.1. Game loop

- Фиксированный шаг `dt = 1/60` с аккумулятором, кадр ограничен сверху 0.1 с (нет «spiral of death»).
- Рендер переменный, интерполяции между тиками нет — состояние отрисовывается как есть.

### 3.2. Состояния

```
MENU ──(клик/Space/Enter)──▶ PLAYING ──(Esc/P)──▶ PAUSED ──(Esc/P/Space/Enter)──▶ PLAYING
                                │
                                ├──(волна закончена)──▶ SHOP (draft) ──(выбрал карту / skip)──▶ PLAYING …
                                │
                                └──(HP ≤ 0)──▶ GAME_OVER ──(любое действие)──▶ MENU
```

**Глобальные хоткеи:** `M` — mute (любое состояние), `Esc`/`P` — пауза (только в `PLAYING`).

### 3.3. Модули

| Файл | Ответственность |
|---|---|
| `main.js` | Bootstrap: Renderer, Input, Game |
| `game.js` | Стейт-машина, волновой прогресс, Game loop |
| `player.js` | Игрок: движение, HP, атака, анимации |
| `caravan.js` | Корованы, стражники, AI, спавн волн |
| `combat.js` | Разрешение атак, формула урона, стрелы |
| `loot.js` | Монеты, магнит, сбор |
| `world.js` | Террейн, дорога, декорации (кешируется в offscreen canvas) |
| `camera.js` | Смуз-фоллоу, трясучка, world↔screen |
| `particles.js` | Пыль, искры, всплески крови, золотые блёстки |
| `renderer.js` | Примитивы 2D Canvas |
| `ui.js` | HUD, меню, магазин, экран смерти |
| `audio.js` | Web Audio синтез (saw/sine/square/noise) |
| `input.js` | Клавиатура, мышь, тач |
| `utils.js` | Vec2, коллизии, `CONST`, seeded random |

## 4. Мир

- Размеры: **2400 × 1600** px, плитка 32 px.
- Террейн: 3 типа тайлов (песок, дорога, кромка дороги) с вариативной палитрой через seeded random (seed 42).
- Дорога: 8 сегментов по ширине, контрольные точки с отклонением ±20% по Y, интерполяция Catmull-Rom, ширина ≈ 2.5 тайла.
- Декорации (seed 123, 120 объектов, без пересечений с дорогой):
  - Камни (40%) — 4–12 px, цвет `#8a8278`.
  - Кусты (30%) — круглые, цвет `#6b8e3a`.
  - Кактусы (30%) — «крестовые», случайной высоты.
- Отрисовка террейна закеширована в offscreen canvas — в основной canvas копируется только видимый кусок вьюпорта.

**Палитра (`CONST.COLOR_*`):** песок `#e8c872 / #d4b45c / #f0d888`, дорога `#c4a24e / #b0903a`, золото `#ffd700`, игрок `#c0392b`, стражник `#2c3e50`, корован `#8b6914`, HP-бар `#e74c3c`.

## 5. Камера

- Экспоненциальный follow к позиции игрока, коэффициент сглаживания **8.0**.
- Клампинг в границы мира.
- Трясучка (интенсивность × длительность):
  - удар игрока — 3 × 0.10 с,
  - игрока ударили в ближнем бою — 4 × 0.15 с,
  - попадание стрелы — 3 × 0.12 с.

## 6. Игрок

| Параметр | Значение |
|---|---|
| Max HP | 100 |
| Скорость | 160 u/s |
| Ускорение | 800 u/s² |
| Трение | 6.0 (экспоненциальное затухание) |
| Радиус | 12 px |
| Базовый урон | 15 |
| Дальность атаки | 28 px (+ радиус = 40 px эффективного досягания) |
| Кулдаун атаки | 0.4 с |
| Пассивная регенерация | 2 HP/сек (накопление дробных значений) |
| Dash speed | 460 u/s |
| Dash duration | 0.18 с |
| Dash i-frames | 0.22 с (чуть длиннее dash) |
| Dash cooldown | 1.0 с |

**Рывок (Dash):**
- Запуск по `Shift`. Направление берётся из текущего вектора движения; если стоим — из facing.
- Во время рывка ввод движения игнорируется (нельзя «повернуть» в полёте), friction не применяется, velocity = `dashDir × 460`.
- `iframeTimer > 0` → `takeDamage()` полностью игнорирует любой урон.
- Визуал: прозрачность игрока падает до 0.55 при активных i-frames, след из светлых частиц `spawnDashTrail`.
- HUD: полоска зарядки с иконкой ⚡ рядом с золотом. Жёлтая когда готова, тёмная пока перезаряжается.
- Звук: `audio.playDash()` — короткий sine-свип 320→120 Гц.

**Атака:** широкая фронтальная дуга около 110°, срабатывает в момент атаки по всем врагам в `range + radius`. Минимальный урон в ближнем бою — 1 (не бывает «нулевых» ударов).

**Формула брони:**
```
effective = max(1, round(base × 100 / (100 + armor)))
```
Пример: 15 урона по стражнику с бронёй 30 → `15 × 100/130 ≈ 11.5 → 12`.

**Анимации:**
- `IDLE` — статичный спрайт.
- `WALK` — 2 кадра по 0.15 с, зеркалится по направлению.
- `ATTACK` — 0.2 с, squash/stretch 1.3× по X, 0.75× по Y с возвратом к 1.0.
- Flash белым при получении урона (0.1 с).

## 7. Корованы

| Тип | Скорость | Loot (gold) | Охрана (база) | HP | Название |
|---|---|---|---|---|---|
| `DONKEY` | 40 | 10–25 | 1 | 30 | Осёл |
| `WAGON` | 30 | 30–60 | 2 | 60 | Телега |
| `ROYAL` | 20 | 80–150 | 3 | 100 | Карета |

- Движутся по сплайну дороги через параметр `pathT ∈ [0, 1]`. Вышли за границы — считаются сбежавшими.
- При смерти или уничтожении всей охраны — рассыпают loot на **3–8** монет (разлёт 40–100 u/s, трение 5.0).
- Спрайт масштабируется ×2; **босс** масштабируется ×3 и получает золотую корону сверху.

### 7.1. Босс-корованы

- Появляются каждую **5-ю волну**.
- Всегда `ROYAL`.
- `BOSS_HP_MULTIPLIER = 3` → 300 HP.
- `BOSS_LOOT_MULTIPLIER = 3` → 240–450 gold.
- `BOSS_GUARD_COUNT = 5` (фикс, перекрывает масштабирование).

## 8. Стража

Три архетипа, HP-бар окрашивается по типу.

| Параметр | Basic | Armored | Archer |
|---|---|---|---|
| HP | 40 | 90 | 30 |
| Урон | 8 | 12 | 6 (стрела) |
| Скорость | 80 | 55 | 65 |
| Detection | 120 | 100 | 180 |
| Chase | 200 | 200 | 200 |
| Attack range | 20 | 20 | 150 |
| Attack cd | 0.8 с | 0.8 с | 1.5 с |
| Броня | 0 | 30 | 0 |
| Цвет HP | red | blue | green |

Общие параметры: радиус 10 px, 2-кадровая анимация ходьбы (по 0.2 с), flash белым при уроне, HP-бар появляется только когда раненый.

### 8.1. Композиция по волнам

- Волны 1–3: только `BASIC`.
- Волны 4–5: 65% `BASIC` + 35% `ARMORED`.
- Волны 6+: 40% `BASIC` + 35% `ARMORED` + 25% `ARCHER`.

### 8.2. AI стражи

Стейт-машина: `PATROL → CHASE → RETURN`.

- **PATROL:** орбитирует вокруг корована с индивидуальным офсетом ±40 px по X/Y.
- **CHASE:** преследует игрока при попадании в `detection`; выходит из chase, если игрок дальше `chase range` или корован уехал > 1.5× chase.
- **RETURN:** возвращается к коровану; повторное обнаружение игрока снова включает chase.
- Если корован мёртв — стражник залочен в `CHASE` навсегда.

**Ближник (Basic/Armored):** подходит на дистанцию `enemy.r + player.r + 4 ≈ 26 px` и держит «standoff», чтобы не слипаться с игроком.

**Лучник (Archer):** держит предпочтительную дистанцию 100 px (зона 70–130 px). Ближе 70 — отступает, дальше 130 — идёт вперёд, в зоне — стрейфит. Стреляет только в `CHASE` в пределах attack range.

### 8.3. Стрелы

- Скорость 200 u/s, радиус 4 px, lifetime 3 с.
- Визуально: коричневая линия + тёмный наконечник длиной 8 px.
- Урон фиксированный (урон лучника), коллизия по кругу.

## 9. Бой и лут

### 9.1. Разрешение атаки

- Игрок нажал атаку → проверяется круг `(player.pos, range + radius)` против всех стражников и корованов.
- Всем найденным целям наносится урон с учётом брони.
- Спавнятся искры, trembling камеры, слэш-эффект.

### 9.2. Слэш-эффект

- Длительность 0.18 с, две концентрические дуги (внешняя белая 5 px, внутренняя жёлтая 2 px).
- Разворачивается от 0° до 110° вокруг направления атаки.

### 9.3. Лут

- `LOOT_RADIUS = 6`, `LOOT_MAGNET_RANGE = 40`, `LOOT_MAGNET_SPEED = 200`.
- Монеты разлетаются с трением 5.0, слегка покачиваются (sin-бобинг ±2 px).
- При входе в радиус магнита тянутся к игроку на 200 u/s.
- Собираются при касании (радиус игрока + монеты = 18 px).

## 10. Прогрессия волн

- Обычное число корованов на волне: `1 + floor(wave / 2)`, максимум **6**.
- Каждая 5-я волна — **босс + обычные** (на wave 5 это 1 босс + 1 обычный).
- Дополнительная охрана: `+1 стражник на 3 волны` к базовому количеству каравана.
- Состав корованов:
  - Волны 1–2: только `DONKEY`.
  - Волны 3–4: 60% `DONKEY`, 40% `WAGON`.
  - Волны 5+: 30% `DONKEY`, 40% `WAGON`, 30% `ROYAL`.

**Условие окончания волны:** все корованы убиты или сбежали **и** весь лут подобран.

### 10.1. Flawless-бонус

`FLAWLESS_WAVE_BONUS = 50`. Если за волну игрок:
- не получил урона,
- ограбил хотя бы один корован,

то к счёту добавляется `50 × wave` (wave 5 → +250).

## 11. Магазин — Draft карточек

У магазина два входа:
1. **Автоматически между волнами** (free-режим) — обязательный выбор одной из **5 карт**, picks бесплатны.
2. **Физический магазин в пустыне** (paid-режим) — игрок подходит к домику и нажимает `E`, picks стоят золота.

### 11.1. Физический магазин

- Деревянный домик с крышей, окнами и вывеской-монеткой. Фиксированная позиция `(worldW/2 + 150, worldH/2 − 140)` — северо-восточнее спавна, хорошо виден при старте волны.
- Радиус взаимодействия `56 px`. Пока игрок в радиусе — над домиком пульсирует подсказка `[E] ТОРГОВАТЬ`.
- Вход: `E` в `PLAYING`. Выход: кнопка «Выйти из шопа», `Esc`, `E` или `Enter` — возвращается в `PLAYING` без смены волны.
- Оффер шопа **персистентен в рамках одной волны** (сохраняется между визитами), чтобы нельзя было реролить через выход/вход. Refresh на старте новой волны.

### 11.2. Стоимость карт (paid-режим)

По редкости:

| Редкость | Цена |
|---|---|
| Common | 15 g |
| Uncommon | 40 g |
| Rare | 80 g |

Free-режим игнорирует стоимость. Реролл работает в обоих режимах — 10 g + 2 g за каждый прошлый реролл.

### 11.1. Пул карточек

Карточки взвешены по редкости: **common 60**, **uncommon 30**, **rare 10**.

**Common (статы, stackable):**

| Карта | Иконка | Эффект |
|---|---|---|
| Точный удар | ⚔ | +5 к урону |
| Крепкий череп | ❤ | +25 к max HP и +25 HP |
| Лёгкая походка | 👢 | +20 к скорости |
| Длинная рука | 🗡 | +4 к радиусу атаки |
| Быстрый замах | ⏱ | −12% к кулдауну атаки (пол 0.12 с) |

**Uncommon (механики, stackable):**

| Карта | Иконка | Эффект |
|---|---|---|
| Вампиризм | 🩸 | +10% HP от нанесённого урона за свинг |
| Жадные руки | 🧲 | +50% к радиусу магнита лута |
| Шипы | 🌵 | Ближники получают 25% урона в ответ |
| Второе дыхание | 💚 | +2 HP/сек к регенерации |
| Быстрые ноги | ⚡ | −25% к кулдауну рывка |

**Rare:**

| Карта | Иконка | Эффект | Stackable |
|---|---|---|---|
| Берсерк | 💀 | +12 урона, −15 max HP | ✔ |
| Круговой удар | 🌀 | Атака бьёт во все стороны (полный 360°) | ✘ (уникальная) |

### 11.2. Механика драфта

- **Pick:** клик по карте → применяется, магазин закрывается, старт следующей волны.
- **Skip:** пропустить волну без карты (клик/Enter).
- **Reroll:** перекинуть 3 новые карты за золото. База **10 g**, рост **+2 g** за реролл в пределах одного магазина.
- Unstackable-карты (например, «Круговой удар») исчезают из пула после первого взятия.
- Карты раскидываются без повторов в пределах одного драфта.

Игрок также сохраняет золото как метрику счёта — оно всё ещё добавляется к итоговому score.

## 12. Счёт, Game Over и рекорды

- Счёт = сумма собранного золота + flawless-бонусы.
- HUD показывает: волну, счёт, здоровье, количество оставшихся корованов и стражников, индикатор mute при выключенном звуке.
- Game Over при `HP ≤ 0`. Экран итогов: волна, ограбленных корованов, общий счёт, лучший рекорд.
- **Лучший счёт и волна** сохраняются в `localStorage` через `js/storage.js`. Если в текущем забеге побит рекорд — на экране смерти появляется «НОВЫЙ РЕКОРД!». Главное меню тоже показывает текущий лучший результат.
- **Пауза:** `Esc`/`P` замораживают игру с оверлеем «ПАУЗА». Любая из клавиш `Esc`/`P`/`Space`/`Enter` снимает паузу.
- **Mute:** `M` глобально переключает звук и сохраняет предпочтение в `localStorage`.

## 13. Эффекты и «сочность»

**Частицы:**
- **Пыль под движением** — раз в 0.08 с при `|v| > 20`, 1–2 шт., цвет песка, гравитация 30.
- **Искры удара** — 5–8 шт., белые/золотые, радиальный разлёт 60–160 u/s.
- **Золотая пыль при сборе** — 6–10 шт., лёгкий подбрасывающий импульс.
- **Кровавый всплеск при смерти врага** — 8–13 красных частиц, гравитация 60.

**Визуальные фидбэки:**
- Flash белым при уроне (0.1 с) — у игрока и у стражи.
- Squash/stretch на атаке/ударе.
- Плавающие числа урона (белые при хитах, золотые при пикапе, зелёные на flawless).
- Трясучка камеры на ударах.

## 14. Аудио

Все звуки синтезируются в рантайме через Web Audio API. Мастер-громкость 0.3.

| Событие | Осциллятор | Частота | Длительность | Gain |
|---|---|---|---|---|
| Атака | saw | 200 → 80 Гц | 0.15 с | 0.15 |
| Удар попал | square | 150 → 40 Гц | 0.10 с | 0.20 |
| Подбор монеты | sine | 600 → 900 → 1200 Гц | 0.20 с | 0.12 |
| Смерть стражника | saw | 300 → 50 Гц | 0.35 с | 0.12 |
| Игрок ранен | square | 200 → 100 → 150 Гц | 0.20 с | 0.15 |
| Ветер (ambient) | white noise + LP 200 Гц | loop | — | 0.04 |

## 15. UI

**HUD:** полоса высотой 40 px сверху — HP-бар, золото, волна, счёт, счётчик корованов/стражников.
**Главное меню:** тёмный оверлей `#1a1a2e`, пульсирующий заголовок, мигающий prompt, декоративные скрещённые мечи.
**Магазин:** карточки апгрейдов с иконкой, названием (русский), описанием, ценой, счётчиком покупок; кнопка «Следующая волна» (также на `Enter`).
**Game Over:** заголовок «КОНЕЦ ИГРЫ», финальная статистика, prompt возврата в меню.

## 15a. Телеметрия сессий

Для анализа баланса каждая сессия пишется в `localStorage` через `js/session-logger.js`. Это чисто локальные данные, никуда не отправляются.

### 15a.1. Что хранится в сессии

- `id`, `commit` (из `version.json`), `startedAt`, `endedAt`, `durationMs`.
- `meta` — метаданные запуска (например, build info).
- `events[]` — хронологический список событий с таймстемпом `t` (мс от старта):
  - `session_start`, `wave_start`, `wave_end`
  - `guard_killed` (type, wave), `caravan_robbed` (type, value, wave, boss)
  - `player_damaged` (amount, source, hpAfter, wave)
  - `card_picked` (id, rarity, mode, cost, wave), `reroll`
  - `shop_opened` / `shop_closed` (origin: wave|world)
  - `dash`, `flawless`, `player_died`
- `summary` — агрегаты: `waveReached`, `wavesCompleted`, `guardsKilled{,ByType}`, `caravansRobbed{,ByType}`, `bossesDefeated`, `damageDealt`, `damageTaken`, `goldEarned`, `goldSpent`, `attacks`, `dashesUsed`, `rerolls`, `cardsPicked[]`, `flawlessWaves`, `died`, `finalScore`, `durationMs`.

### 15a.2. Хранение и экспорт

- Кольцевой буфер на 100 последних сессий в ключе `korovany.sessions`.
- В главном меню показывается «Логов сессий: N».
- **Клавиша `L`** — скачать все сессии JSON-файлом `korovany-sessions-<timestamp>.json`.
- **`Shift+L`** — очистить все сохранённые сессии.
- **`Shift+U`** — синхронизировать все сессии из localStorage на telemetry-сервер (бэкфилл после оффлайн-плейтеста).
- Из devtools доступно как `window.__game.logger` — можно дергать `buildExportJson()`, `endSession()` и т.п.

#### Онлайн-сборщик

- `scripts/telemetry-server.js` — минимальный Node stdlib HTTP-сервер (порт 12000), принимает `POST /sessions` и пишет файлы в `telemetry/sessions/<id>.json` (идемпотентно по `id`).
- Публикуется наружу через ngrok: `ngrok http --url=rapid-mayfly-intense.ngrok-free.app 12000`.
- В `js/session-logger.js` константа `TELEMETRY_URL` указывает на ngrok-туннель. После `endSession()` клиент делает fire-and-forget `fetch` с `keepalive: true`. Ошибки глушатся — localStorage остаётся источником истины.
- Аплоад выполняется **только из браузера** (`typeof window !== 'undefined'`), чтобы тесты и headless sim не засоряли сток.
- Переопределение URL в рантайме: `window.KOROVANY_TELEMETRY_URL = 'http://localhost:12000/sessions'` (удобно для локального дебага).
- Опциональная защита от случайного мусора из интернета: запуск сервера с `TELEMETRY_TOKEN=...`, тот же токен прописать в `TELEMETRY_TOKEN` в `session-logger.js`.

### 15a.3. Commit hash

- `version.json` в корне проекта содержит `commit` и `short` — текущий build tag.
- `main.js` подгружает его асинхронно при старте (`fetch('./version.json')`) и прокидывает в `Game → SessionLogger`, чтобы каждая записанная сессия была привязана к конкретной ревизии.
- Обновление: `./scripts/update-version.sh` — записывает текущий `git HEAD` в `version.json`. Можно повесить в `post-commit` hook: `ln -sf ../../scripts/update-version.sh .git/hooks/post-commit`.

### 15a.4. Анализ

Экспортированный JSON можно парсить любым инструментом. Типовые запросы:

- **Какая карта чаще приводит к смерти?** — группировать сессии с `died=true` по последним взятым карточкам.
- **Где игрок теряет HP?** — сумма `player_damaged.amount` по `source` и `wave`.
- **Баланс волн:** среднее время `wave_end.durationMs` по номеру волны, % flawless, % escaped.
- **Сравнение коммитов:** group by `commit`, сравнить медианную `waveReached`.

## 15b. Симуляционный фреймворк

Для анализа баланса без живого тестирования в `js/sim/` лежит headless-симулятор: тот же `World`, `Player`, `Guard`, `Caravan`, `Projectile`, `Loot`, `performAttack`, `spawnWave` — только без `Renderer`, `Camera`, `Input`, `GameAudio`, `ParticleSystem`. Entity-классы не трогают DOM, поэтому переиспользуются 1:1.

### 15b.1. Архитектура

```
AIPolicy (interface)
  ├─ GreedyPolicy      — бьёт ближайшего стража, иначе гонится за корованом, иначе лут, первая карта
  ├─ RandomCardPolicy  — та же навигация, random-индекс в магазине
  └─ RunnerPolicy      — игнорит стражу, лезет в корованы

Simulator
  ├─ new World(), new Player(), new UI() (только для пула карт), new SessionLogger()
  ├─ цикл: policy.decidePlaying(view) → SimInput → Player.update → attack → caravans/guards/projectiles/loot → wave_end
  ├─ на wave_end: ui.beginFreeDraft() → policy.decideShop(offer) → ui.pickCard / tryReroll / skip
  ├─ writes events via SessionLogger (те же хуки что в live-игре)
  └─ returns finished session object на смерти / max waves

runBatch(policyFactory, N, opts)  — гонит N сессий подряд
summarizeBatch(sessions)          — агрегаты (min/median/mean/max по волнам и счёту, pickCounts по карточкам)
```

### 15b.2. Policy API

```js
class AIPolicy {
  name;
  decidePlaying(view) → { moveX, moveY, attack, dash }
  decideShop(view)    → { action: 'pick' | 'reroll' | 'skip', index? }
}
```

`view` в `decidePlaying`:
- `wave`, `player` (pos, hp, maxHp, gold, stats, таймеры, модификаторы)
- `caravans`, `guards`, `projectiles`, `loots` — реальные entity-объекты (read-only)
- `shop` — ссылка на `Shop` сущность

`view` в `decideShop`: `{ wave, mode, offer, player, gold }`.

### 15b.3. CLI

```
node js/sim/run.js [--policy name] [--count N] [--max-waves N] [--out file] [--quiet]
```

Пример вывода (`--policy greedy --count 30 --max-waves 20`):
```

  attackRange      ████████████████████ 12
  speed            ████████████████████ 12
  lifesteal        ██████████████████·· 11
  ...
```

Флаг `--out file.json` пишет полные сессии в JSON (та же схема, что и экспорт живых сессий из меню) — можно загонять в jq/pandas для более глубокого анализа.

### 15b.4. Текущие наблюдения

Первые прогоны (commit `9270b06`):

- **Greedy-бейзлайн** хронически умирает на волне 4 (медиана = 4, max = 5). Это «стена сложности» для дурного AI.
- **Runner** (игнорит стражу) уступает greedy — средняя волна 3.1 vs 4.2. Значит стража не декоративна, её нельзя пропускать.
- **Random-cards** vs **Greedy**: max score 588 vs 208 — «первая карта всегда» явно недооптимально, лотерея иногда собирает лучший билд.

Сами цифры подсказывают, что либо волна 4 слишком скачкообразно тяжёлая, либо базовый урон игрока (15) не поспевает за HP брони/лучников, либо greedy-heuristic слишком прост. Для анализа — прогонять тот же policy после каждого изменения баланса и смотреть, как сдвигается медианная волна.
